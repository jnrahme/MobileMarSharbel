- Use the `jnrahme` GitHub CLI account for pushes to `origin` in this repo.
- Before any push, run `gh auth switch -u jnrahme` and confirm `jnrahme` is the active account.
- After each completed phase, push the current `main` branch with `git push origin main` unless the user explicitly requests a different branch or PR flow.
- After the push completes, switch the active GitHub CLI account back to `joey-rahme_boats` with `gh auth switch -u joey-rahme_boats`.
- If a direct push to `main` is rejected, stop and report the exact rejection instead of changing the branch strategy silently.
